## P1 — الدورة القادمة

### F1-1 — الرصيد الدائن للعميل واستخدامه (Overpayment → Credit)

**المصدر:** BUG-04 + tasks-bugs-and-features.md (5.4 + 5.5)

**الوصف التفصيلي:**

عندما يدفع العميل أكثر من قيمة الفاتورة، يجب أن:
1. يُضاف الفائض لرصيد العميل الدائن `creditBalance`
2. في فاتورة جديدة: يظهر رصيد العميل مع خيار "استخدام الرصيد المتاح"
3. زر **"صالح كل المستحقات"** لتسوية جميع الفواتير غير المدفوعة لعميل واحد

**الملفات المتأثرة:**
- `src/types/index.ts` — إضافة `creditBalance?: number` للـ Customer
- `src/store/AppContext.tsx` — تعديل `recordSalesReceipt` + إضافة `settleAllDues`
- `src/pages/SalesInvoiceNewPage.tsx` — عرض رصيد العميل وخيار الاستخدام
- `src/pages/SalesInvoiceDetailPage.tsx` — زر "صالح كل المستحقات"

**الجهد المقدّر:** 3-5 أيام
**القيمة:** عالية — يمنع ضياع أموال العملاء

### F1-2 — إنشاء المرتجع من الفاتورة مباشرة

**المصدر:** tasks-bugs-and-features.md (5.6)

**الوصف:**
في صفحة تفاصيل الفاتورة، زر **"إنشاء مرتجع"** يفتح Dialog فيه أصناف الفاتورة الأصلية جاهزة للاختيار منها.

**السلوك المقترح:**
```
المستخدم يضغط "إنشاء مرتجع" من SalesInvoiceDetailPage
  ↓
يفتح SalesReturnDialog مُملَّأ مسبقاً بـ:
  - lines من الفاتورة الأصلية
  - كمية = صفر لكل سطر (المستخدم يختار ما يرده)
  - سعر = سعر الفاتورة الأصلية (غير قابل للتعديل)
  - أقصى كمية = الكمية الأصلية
```

**الملفات المتأثرة:**
- `src/features/returns/SalesReturnDialog.tsx` — قبول `initialLines` prop
- `src/pages/SalesInvoiceDetailPage.tsx` — إضافة زر + تمرير البيانات
- نفس الفكرة لـ `PurchaseReturnDialog` + `PurchaseInvoiceDetailPage`

**الجهد المقدّر:** 2-3 أيام
**القيمة:** عالية — يُقلل أخطاء إدخال المرتجعات

### F1-3 — صفحة المستحقات القادمة (Receivables Page)

**المصدر:** tasks-bugs-and-features.md (6.2)

**الوصف:**
صفحة أو تبويب جديد يعرض:
- جميع الفواتير الآجلة غير المدفوعة
- مرتبة تصاعدياً بتاريخ الاستحقاق
- مع تمييز المتأخرة (تجاوزت تاريخ الاستحقاق) بلون أحمر

```typescript
// منطق الفلتر
const pendingReceivables = salesInvoices
  .filter(inv => inv.status !== "paid" && inv.paymentType === "account" && !inv.cancelled)
  .sort((a, b) => (a.paymentDueDate ?? "9999").localeCompare(b.paymentDueDate ?? "9999"));
```

**الملفات المتأثرة:**
- إضافة `src/pages/ReceivablesPage.tsx` (أو قسم في `ReportsPage.tsx`)
- `src/App.tsx` — إضافة المسار
- `src/components/layout/Sidebar.tsx` — إضافة الرابط

**الجهد المقدّر:** 1-2 يوم
**القيمة:** عالية — رؤية مالية أساسية
