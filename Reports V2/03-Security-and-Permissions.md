أهم تطور منذ V1: **بوابة الجلسات على التخزين** أصبحت كاملة، ومسار `btoa` الضعيف أُزيل نهائياً، وسجل التدقيق أصبح موجوداً. المتبقي بندان متوسطان (صلاحية مسار التعديل + جلسة الطباعة) وتحسينات اختيارية.

## نقاط القوة الحالية ✅

### 1. بوابة الجلسات على كل عمليات التخزين (تحسّن جديد عن V1)

```javascript
// main.cjs:1246-1255
function canReadRendererStorage(event) {
  return !ownerExistsInStore() || Boolean(getSession(event));   // القراءة تتطلب جلسة بعد وجود مالك
}
function canMutateRendererStorage(event, key) {
  if (!ownerExistsInStore()) return false;
  if (!getSession(event)) return false;
  if (String(key) === `${STORE_PREFIX}users`) return hasOwnerSession(event); // كتابة المستخدمين للمالك فقط
  return true;
}
```

- `storage:export` و`storage:import` و`storage:clear-prefix` و`backup:write-file` كلها **owner-only**.
- كتابة مفتاح `users` تمرّ بـ `mergeRendererUsersValue` الذي يرفض أي passwordHash غير argon2 — لا يمكن حقن مستخدم بهاش ضعيف من الـ renderer.
- هاشات كلمات المرور تُخفى عن الـ renderer (`[REDACTED]`) وعن كل النسخ الاحتياطية.

### 2. المصادقة وكلمات المرور
- argon2id (64MB / t=3 / p=4) في العملية الرئيسية فقط، مع dummy-hash لمقاومة timing attacks.
- Rate limiting للدخول (5 محاولات/60 ث) ولكود الدعم (5/10 دقائق) في طبقة main — لا يُتحايل عليه.
- fallback الويب الآن SHA-256 فقط، **ومسار btoa القديم يُرفض** (`auth.ts:21-22`) — SEC-02/BUG-11 من V1 مُغلقان.
- استعادة كلمة مرور المالك بكود دعم موقّع ومرتبط بالجهاز مع rate limit.

### 3. الترخيص والتشفير
- Ed25519 + canonical JSON + ربط بالجهاز + كشف إرجاع الساعة (`clock_tampered`).
- SQLCipher بمفتاح مشتق من بصمة الجهاز — القاعدة لا تعمل منقولة لجهاز آخر.

### 4. عزل العملية والتحزيم
- contextIsolation + sandbox + nodeIntegration:false + webSecurity.
- CSP صارم في الإنتاج، Electron Fuses الستة مفعّلة، ASAR + integrity validation.
- نافذة الطباعة: `will-navigate` ممنوع + `setWindowOpenHandler: deny`.

### 5. سجل التدقيق (SEC-03 من V1 — مُغلق)
15 نوع عملية حساسة (حذف/إلغاء فواتير، تسويات مخزون، حذف كيانات، حركات خزنة يدوية) مع المستخدم والوقت — صفحة owner-only.
⚠️ لكنه **غير مشمول بالنسخ الاحتياطي** (V2-B04) وسقفه 1000 قيد.

## الثغرات والملاحظات المفتوحة

### SEC-V2-01 🟠 — مسار تعديل فاتورة المبيعات بدون صلاحية edit

(= V2-B06 في تقرير الأخطاء — مذكور هنا لأنه ثغرة صلاحيات.)
[`App.tsx:159-166`](../src/App.tsx#L159): المسار `/sales/:id/edit` محمي بـ view فقط، والصفحة لا تتحقق داخلياً. موظف عرض-فقط يستطيع تعديل فاتورة (وبسبب V2-B02 يغيّر المُحصَّل دون أثر في الخزنة!). **الإصلاح سطر واحد.**

### SEC-V2-02 🟡 — `print:route` و`print:save-current-pdf` بلا فحص جلسة

[`main.cjs:1478`](../electron/main.cjs#L1478): توليد HTML الفاتورة يتم في main مباشرة من القاعدة، فأي renderer غير مصادق (نظرياً) يطلب طباعة أي فاتورة وحفظها PDF. الخطر العملي منخفض (التطبيق kiosk-like) لكنه كسر لنموذج "كل IPC حساس خلف جلسة". إضافة `getSession(event)` للمعالجات الأربعة `print:*`.

### SEC-V2-03 🟢 — مفتاح تشفير القاعدة قابل للاشتقاق على نفس الجهاز

كما في V1 (SEC-01): `sha256(APP_SALT + machineId)` — قيد تصميمي مقصود لتطبيق أوف-لاين. مقبول. التعزيز الاختياري (دمج سر من كلمة مرور المالك) مقايضته فقدان البيانات عند النسيان — غير موصى به حالياً.

### SEC-V2-04 🟢 — حجب DevTools تجميلي

كما في V1 (SEC-04): الحماية الفعلية من الـ Fuses. يمكن إضافة `app.commandLine.appendSwitch("remote-debugging-port", "0")` كتحسين.

## توصيات أمنية إضافية (غير عاجلة)

1. **شهادة توقيع كود حقيقية (OV/EV)** بدل الذاتية — تقلل إنذارات SmartScreen عند العملاء وتحمي سمعة المنتج. (تكلفة سنوية، قرار تجاري.)
2. **سقف سجل التدقيق**: رفعه أو أرشفته دورياً لملف بدل القصّ عند 1000 — مع شموله بالنسخ الاحتياطي (V2-B04).
3. **قفل الجلسة بعد خمول** (مثلاً 15 دقيقة بلا نشاط → شاشة قفل) — مناسب لجهاز كاشير مشترك.
4. **تحقّق سلامة النسخة الاحتياطية**: إضافة checksum (sha256) داخل ملف الـ backup والتحقق عند الاستيراد لاكتشاف الملفات التالفة/المعدّلة.
